# Dashboard Brief

Do not hardcode a crowded dashboard. Let the implementation evolve.

The local dashboard should help answer these questions:

1. Is inflation above or below the selected baseline?
2. Is the transitory component rising or falling?
3. Is the current reading historically elevated?
4. How persistent does the model estimate the current regime to be?
5. How sensitive are conclusions to baseline and window choices?
6. Are market linkages descriptive, predictive, or not robust?

## Preferred dashboard behavior

- clear labels for baseline and sample period
- paper replication and live-safe modes separated
- concise interpretation panels
- downloadable tables only after values are validated
- visual choices made for clarity, not because a prompt prescribed them

## Visualization freedom

Claude may choose suitable chart types. The only requirement is that visuals should make the macro interpretation easier and should not hide methodology assumptions.
